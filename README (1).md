## Overview

DIALECTA is a full-stack AI debate simulator built with **Flask** and the **Anthropic Claude API**. Two independent AI agents — ADVOCATE (Pro) and CHALLENGER (Con) — are automatically assigned opposing positions on any topic you provide. They debate across structured rounds, and every argument is scored and analyzed in real time for logical fallacies, bias, and argument strength.

The system is designed as an educational tool for **critical thinking and logical reasoning**. The winner is determined by the quality of their arguments — not their opinion — using a transparent, explainable scoring model.

## Features

### Core Debate System

- **Two autonomous agents** debating independently: ADVOCATE (Pro) and CHALLENGER (Con)
- **Structured round format**: Opening → Rebuttal → Cross-Argument → Closing
- **Configurable tone**: Academic, Formal, Aggressive, or Socratic
- **2 to 4 debate rounds** selectable before starting
- **Topic suggestions** with one-click fill for quick demos

### Argument Structure

Every argument generated by each agent includes four components:

| Component | Description |
|-----------|-------------|
| **Claim** | The core thesis or assertion (1-2 sentences) |
| **Reasoning** | Logical explanation of why the claim is true (2-3 sentences) |
| **Evidence** | Examples, historical facts, or data supporting the claim |
| **Validity** | Plain English explanation of why the argument is logically sound |

### Real-Time Analysis

Each argument is evaluated on three dimensions (scored 0-100):

- **Logical Score** — internal consistency and deductive soundness
- **Clarity Score** — how clearly the argument is communicated
- **Evidence Score** — strength and relevance of supporting evidence

The **overall score** is the average of these three, displayed as a strength badge: **Strong** (75+), **Moderate** (50-74), or **Weak** (below 50).

### Fallacy Detection

The system flags logical fallacies in plain English, including:

- Strawman argument
- Ad hominem attack
- False cause / post hoc
- Slippery slope
- Appeal to authority
- Hasty generalization
- False dilemma

Each detected fallacy includes an explanation of why it is a fallacy and how it appeared in the argument.

### Bias Analysis

Arguments are tagged if they exhibit:

- **Emotional language** — appeals to feelings over facts
- **Exaggeration** — overstated claims beyond what evidence supports
- **One-sided framing** — ignoring counterfactual evidence
- **Loaded language** — charged words that presuppose a conclusion

### Scoring and Winner Determination

A **fallacy penalty of -5 points** is applied per detected fallacy to the agent's cumulative score. The winner is declared based on penalty-adjusted total score — not based on which position is "correct."

## Getting Started

### Prerequisites

- Python 3.9 or higher
- An [Anthropic API key](https://console.anthropic.com/)

### 1. Extract the Project

```bash
unzip dialecta.zip
cd dialecta
```

### 2. Create a Virtual Environment (Recommended)

```bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Your Environment

```bash
cp .env.example .env
```

Open `.env` and set your values:

```env
ANTHROPIC_API_KEY=sk-ant-...your-key-here...
SECRET_KEY=pick-any-random-string-for-sessions
```

### 5. Run the Application

```bash
python app.py
```

Open **http://localhost:5000** in your browser.

## How It Works

### Debate Flow

```
User submits topic
        |
        v
Flask creates debate record --> saved to debates.json
        |
        v
Browser navigates to /debate
        |
    [For each phase: opening / rebuttal / cross / closing]
        |
        |-- 1. Frontend calls POST /api/generate-argument (agent=pro)
        |-- 2. Flask builds prompt and calls Claude API
        |-- 3. Claude returns structured JSON argument
        |-- 4. Flask scores, saves, returns to browser
        |-- 5. Browser renders argument card live
        |
        |-- 6. Repeat steps 1-5 for agent=con
        |
        v
Frontend calls POST /api/finalize-debate
        |
        v
Flask calculates winner (scores minus fallacy penalties)
        |
        v
Browser navigates to /results?id=<debate_id>
        |
        v
Results page fetches full record and renders charts
```

### AI Agent Architecture

Each AI agent is stateless and modular. For every argument, a fresh Claude API call is made with a system prompt that specifies:

- The agent's assigned position (Pro or Con)
- The debate tone (academic / formal / aggressive / socratic)
- The phase-specific instruction (what kind of argument to make)
- The opponent's last claim (used for rebuttal and cross-argument rounds)

The response is required to be valid JSON with fixed keys. Flask validates and scores each response before saving it to the debate record.

### Data Storage

All debate data is stored in `debates.json` as a flat key-value store keyed by debate ID. No database required. The file is created automatically on first run.

Example record structure:

```json
{
  "a1b2c3d4": {
    "id": "a1b2c3d4",
    "topic": "AI will eliminate more jobs than it creates",
    "tone": "academic",
    "phases": ["opening", "rebuttal", "closing"],
    "date": "2025-03-03 14:22",
    "arguments": [...],
    "pro_scores": [78, 81, 74],
    "con_scores": [70, 85, 68],
    "pro_fallacies": 0,
    "con_fallacies": 1,
    "fallacy_log": [...],
    "bias_log": [...],
    "winner_data": {...}
  }
}
```

## API Reference

All endpoints accept and return JSON.

### POST /api/start-debate

Initialize a new debate session.

**Request:**
```json
{
  "topic": "AI will eliminate more jobs than it creates",
  "tone": "academic",
  "rounds": 3
}
```

**Response:**
```json
{
  "debate_id": "a1b2c3d4",
  "phases": ["opening", "rebuttal", "closing"],
  "phase_labels": {
    "opening": "Opening Statement",
    "rebuttal": "Rebuttal",
    "closing": "Closing Statement"
  }
}
```

---

### POST /api/generate-argument

Generate one argument from one agent for the current phase.

**Request:**
```json
{
  "debate_id": "a1b2c3d4",
  "agent": "pro",
  "phase": "rebuttal",
  "opponent_claim": "Automation has always created more jobs than it destroys."
}
```

**Response:**
```json
{
  "claim": "...",
  "reasoning": "...",
  "evidence": "...",
  "validity": "...",
  "logical_score": 82,
  "clarity_score": 78,
  "evidence_score": 75,
  "overall_score": 78,
  "fallacy": null,
  "fallacy_explanation": null,
  "bias_type": "exaggeration",
  "bias_explanation": "...",
  "agent": "pro",
  "phase": "rebuttal",
  "phase_label": "Rebuttal"
}
```

---

### POST /api/finalize-debate

Calculate and store the winner once all rounds are complete.

**Request:**
```json
{ "debate_id": "a1b2c3d4" }
```

**Response:**
```json
{
  "winner_data": {
    "winner": "pro",
    "winner_label": "ADVOCATE (Pro)",
    "reasoning": "The Pro position wins by reasoning quality...",
    "pro_total": 228.0,
    "con_total": 208.0,
    "pro_avg": 76.0,
    "con_avg": 69.3
  },
  "record": {...}
}
```

---

### GET /api/debate/<debate_id>

Retrieve the full debate record including all arguments, scores, fallacy log, and winner data.

## Debate Mechanics

### Scoring Formula

```
Overall Score  =  (Logical Score + Clarity Score + Evidence Score) / 3

Penalty-Adjusted Score  =  Cumulative Score  -  (Fallacies x 5)
```

The winner is the agent with the higher penalty-adjusted score. If the margin is less than 5 points, the debate is declared a **draw**.

### Phase Descriptions

| Phase | Purpose |
|-------|---------|
| **Opening Statement** | Present primary thesis and core arguments |
| **Rebuttal** | Directly counter the opponent's previous argument |
| **Cross-Argument** | Introduce a new angle while dismantling the opponent's framework |
| **Closing Statement** | Summarize why your position has prevailed; no new claims |

## FAQ

**Do I need a database?**  
No. All debate data is stored in `debates.json`, created automatically on first run.

**Which Claude model is used?**  
`claude-opus-4-6`. Change it in `app.py` inside the `generate_argument` function.

**How long does a debate take?**  
Each argument requires one API call. A 3-round debate (6 arguments total) typically completes in 60-90 seconds depending on API response times.

**Is my API key exposed to the browser?**  
No. All Anthropic API calls happen server-side in Flask. The browser never sees your key.

**Can I deploy this publicly?**  
Yes. Set a strong `SECRET_KEY`, use a production WSGI server, and consider replacing `debates.json` with SQLite or Redis for concurrent users.

```bash
# Example with Gunicorn
pip install gunicorn
gunicorn -w 2 -b 0.0.0.0:8000 app:app
```

**Can I run multiple debates simultaneously?**  
Yes — each debate gets a unique ID. For high concurrency, swap the JSON store for a proper database to avoid write conflicts.
